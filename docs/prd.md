Product Requirements Document (PRD)
1. Introduction 👋🏻
This section provides a foundational understanding of the document and the project being developed.

1.1. Purpose: This document aims to define the functional, technical, and workflow specifications for the "Professional AI Headshot Studio" platform. The goal is to ensure the entire development process is properly directed to build an application capable of transforming casual photos into professional headshots.

1.2. Scope: The system boundaries include a responsive web interface, user authentication system, credit management for generation transactions, photo upload process with face validation, AI processing integration, and result history storage (Vault).

1.3. Target Audience: Professionals, job seekers, freelancers, and corporate teams needing high-quality profile photos (for LinkedIn, CVs, company websites) without having to rent a conventional photo studio.

1.4. Document Revision History: * Version 1.0 (March 18, 2026): Initial draft of PRD and ERD.

2. Product Overview 🔎
A brief summary explaining the product as a whole.

2.1. Product Description: An interactive web application that allows users to upload casual selfies or portraits, then uses AI technology to transform them into professional studio-standard headshots. The application provides customization options for clothing styles and backgrounds, along with instant comparison before users download the results.

2.2. Key Features:

User Authentication (Google & LinkedIn OAuth).

Credit Top-up and Deduction System.

Interactive Studio (Upload, Face Validation, Style Selection).

Background Processing with Status Indicators.

Result Preview with Hold-to-Compare feature (Before/After).

Headshot History (Vault) to re-download or regenerate.

3. User Stories 👤
Explains the functional requirements from the user's perspective.

3.1. Authentication & Accounts

User Story: As a new user, I want to register using my Google or LinkedIn account so I can log in instantly without needing to remember a new password.

User Story: As a registered user, I want to see my remaining credit balance in the top navigation so I know when to top up.

3.2. Core Generation (Studio)

User Story: As a user, I want to upload a photo from my device so the AI can use it as a base reference.

User Story: As a professional user, I want to select a clothing style (e.g., Suit, Business Casual) and background (e.g., Modern Office) so the result matches my career image.

User Story: As a user, I want to see clear progress indicators (e.g., "Optimizing Corporate Lighting") so I know the AI is working and has not encountered an error.

3.3. Preview & History

User Story: As a user, I want to press a button to compare the original photo with the AI result so I can see the difference directly.

User Story: As a user who is unsatisfied with the first result, I want to press the Regenerate button so the system creates a new variation from the same photo.

User Story: As an active user, I want to see all the photos I have generated on the History page so I can download them again later.

4. Technical Requirements ⚙️
Outlines the required technology stack and architecture.

4.1. Frontend Stack:

Framework: React Tanstack Start, TypeScript.

Styling: Tailwind CSS 4.0.

State Management: Jotai.

Animation: GSAP (for smooth page transitions and Hold-to-Compare interactions).

4.2. Backend & Database:

API Framework: Hono.

Database: PostgreSQL.

ORM: Prisma.

Storage: R2 / Supabase.

Auth: Better Auth.

AI Gen: Nano Banana 2 / Seedream / FAL.ai.

4.3. Access Control:

Public Access: Landing Page, Pricing, and FAQ pages.

Authenticated Access: Dashboard, Studio Processing, History Vault, and Payments.

6. Development Timeline ⏳
Work plan and completion targets, broken down into phases or sprints.

Sprint 1 (Setup, Auth, & Basic UI):

Goal: Users can register, log in, and view an empty dashboard.

Key Activities: Initialize Next.js repository, configure Tailwind 4.0, integrate OAuth, design Landing Page and main Dashboard layout.

Sprint 2 (Core Studio & AI Integration):

Goal: Users can upload photos and the system successfully processes them through a third-party API.

Key Activities: Build the upload interface, integrate cloud storage, connect the AI Image Generator API, create professional loading state animations.

Sprint 3 (History, Credits, & Polish):

Goal: The system can track user credits, store photo history, and is ready for launch.

Key Activities: Build the database schema for credit transactions, develop the history page, implement Before/After interactions, bug fixing, and deployment.
